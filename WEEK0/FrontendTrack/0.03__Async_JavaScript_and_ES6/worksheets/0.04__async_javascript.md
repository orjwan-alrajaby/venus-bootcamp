## 🧩 Challenge 1 — Callbacks in Sequence

You are given async tasks.

Each task uses `setTimeout`, so they finish after different amounts of time.

Your goal is to run them **in order**:

```txt
task1 → task2 → task3 → task4 → task5
```

Even though some tasks finish faster than others.

```js
const task1 = (cb) =>
  setTimeout(() => {
    const message = 'Task 1 has executed successfully!';
    cb(message);
  }, 3000);

const task2 = (cb) =>
  setTimeout(() => {
    const message = 'Task 2 has executed successfully!';
    cb(message);
  }, 0);

const task3 = (cb) =>
  setTimeout(() => {
    const message = 'Task 3 has executed successfully!';
    cb(message);
  }, 1000);

const task4 = (cb) =>
  setTimeout(() => {
    const message = 'Task 4 has executed successfully!';
    cb(message);
  }, 2000);

const task5 = (cb) =>
  setTimeout(() => {
    const message = 'Task 5 has executed successfully!';
    cb(message);
  }, 4000);

const asyncTasks = [task1, task2, task3, task4, task5];

const executeInSequenceWithCBs = (tasks, callback) => {
  // your code here
};

executeInSequenceWithCBs(asyncTasks, (messages) => {
  console.log(messages);
});
```

👉 Expected output:

```js
[
  'Task 1 has executed successfully!',
  'Task 2 has executed successfully!',
  'Task 3 has executed successfully!',
  'Task 4 has executed successfully!',
  'Task 5 has executed successfully!'
]
```

## 🧩 Challenge 2 — Promises in Parallel

You are given a list of APIs.

Your goal is to fetch all APIs **in parallel**.

Parallel means:

* all requests start at the same time
* the function waits until all of them finish
* the final result should contain all API results

```js
const apis = [
  {
    apiName: 'products',
    apiUrl: 'https://dummyjson.com/products',
  },
  {
    apiName: 'users',
    apiUrl: 'https://dummyjson.com/users',
  },
  {
    apiName: 'posts',
    apiUrl: 'https://dummyjson.com/posts',
  },
  {
    apiName: 'comments',
    apiUrl: 'https://dummyjson.com/comments',
  },
];

const executeInParallelWithPromises = async (apis) => {
  // your code here
};

executeInParallelWithPromises(apis)
  .then((results) => console.log(results))
  .catch((error) => console.log(error));
```

👉 Expected output shape:

```js
[
  {
    apiName: 'products',
    apiUrl: 'https://dummyjson.com/products',
    apiData: {
      // fetched data here
    }
  },
  {
    apiName: 'users',
    apiUrl: 'https://dummyjson.com/users',
    apiData: {
      // fetched data here
    }
  }
]
```

## 🧩 Challenge 4 — Convert Promise Code to Async/Await

Convert this code from `.then()` / `.catch()` to `async/await`.

```js
const getProducts = () => {
  return fetch('https://dummyjson.com/products')
    .then((response) => response.json())
    .then((data) => {
      console.log(data);
      return data;
    })
    .catch((error) => {
      console.log(error);
    });
};
```

👉 Requirements:

* use `async`
* use `await`
* use `try/catch`
* return the fetched data

## 🧩 Challenge 5 — mapAsync

Create a function called `mapAsync`.

It should behave like `.map()`, but it should support asynchronous callback functions.

```js
const mapAsync = async (array, callback) => {
  // your code here
};
```

Test data:

```js
const numbers = [1, 2, 3, 4];

const asyncDouble = (num) => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve(num * 2);
    }, 1000);
  });
};

mapAsync(numbers, asyncDouble).then((result) => {
  console.log(result);
});
```
