## Requirements

### 1. Fetch Movie Data

* Use JavaScript’s `fetch()` to get movie data from the API
* Make sure the data is loaded successfully

---

### 2. Render a Movie List

* Use JavaScript to show movie items in the movie list section
* The movie list should be created dynamically using the fetched data

---

### 3. Default Selected Movie

* When the page loads, one movie should be selected by default
* Show that movie’s details in the details section (according to the Figma design)

---

### 4. Click Behavior

* When the user clicks a movie:

  * that movie should become the selected movie
  * the details section should update

---

### 5. Hover Behavior

* When the user hovers over a movie:

  * the movie should react visually
  * the page may preview that movie’s details

* When the mouse leaves:

  * the page should go back to the selected movie

**Note:**
On touch devices such as phones and tablets, hover does not work in the same way. In that case, click/tap behavior is enough.

---

### 6. Search

* Add a search input
* When the user presses **Enter**, fetch new movie results
* Update the movie list based on the search term

---

### 7. UI States

Your page should handle:

* loading state
* error state
* empty state

For example:

* show `Loading...` while waiting for data
* show a clear message if something goes wrong
* show a clear message if no results are found
